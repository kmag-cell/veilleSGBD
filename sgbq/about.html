Les Base de Données

Une base de données est un système électronique qui permet d'accéder facilement à un ensemble organisé de données, de les manipuler et de les mettre à jour.

Les différentes types de base de données : Base de données

  • hiérarchiques

    • Ce sont les tous premiers programmes de bases de données. Organisées sous forme d’arbre avec ses propres ramifications, les bases de données hiérarchiques permettaient de structurer l’information de manière arborescente

    Chaque enregistrement dépend d’un seul enregistrement et est présenté sous forme d’arbre avec ses ramifications

    En d'autre termes le modèle de hiérarchie est celui où chaque noeud ou composant a une relation enfant parent avec un autre noeud ou composant

  • orientées réseau

    • La base de données réseau est similaire à la base de données hiérarchique mais avec quelques modifications. La base de données réseau permet à l'enregistrement enfant de se connecter à divers enregistrements parents, permettant ainsi des relations bidirectionnelles (c’est à dire plusieurs parents/plusieurs enfants, contrairement à la base de données hiérarchique qui ne comprenait qu’un seul parent).

  • orientées objet

    • Dans la base de données orientée objet, les informations sont stockées à la manière d'un objet.

  • relationnelles

    • C’est le type de bases de données que l’on connaît et que l’on pratique aujourd’hui.

    Basé sur l’algèbre relationnel, il permet de modéliser facilement et sans grosse contraintes les systèmes du monde réel et de créer des bases de données simples à maintenir, à faire évoluer et indépendantes de leur support.Dans ce type de bases de données, les données sont organisées en table

  • non relationnelles

    • Les bases de données non relationnelle ou NoSQL utilise une variété de formats, tels que des documents, des graphiques, de larges colonnes, etc., ce qui offre une grande flexibilité et évolutivité à la conception d'une base de données.

    Les differentes étapes clés pour mettre en place une base de données

  • Identifier le besoin.
  • Définir le modèle de base de données.
  • Choisir le système de gestion de base de données (SGBD)
  • Prévoir son infrastructure.
  • Optimiser sa base de données.
  • Suivre et maintenir la plateforme.
  • Anticiper l'évolution de la plateforme.

    • Avantages et Inconvenients des bases de données

      Avantages
  • Les bases de données font bien plus que contenir des données. Elles aident à résoudre des problèmes, à répondre à des questions et à prendre des décisions. Vous pouvez créer des formules et des calculs pour vous aider dans vos analyses, puis générer rapidement des rapports pour partager votre point de vue
  • Permet à une entreprise de prendre des décisions commerciales plus éclairées.
  • Stockez et récupérez efficacement les informations associées.
  • Aide à analyser et à aggraver les données commerciales.
  • Collectez et stockez des données client cruciales à partir de différentes applications.
  • Fournit des applications opportunes et personnalisées basées sur les données et des analyses en temps réel.
  • Garantit un accès précis, fiable et immédiat aux données commerciales cruciales qui peuvent être utilisées par différents services commerciaux pour comprendre les modèles de données, générer des rapports et prédire les tendances futures.
      • Inconvenients
  • La nature minutieuse, les schémas stricts et les contraintes des bases de données relationnelles rendent quasiment impossible l’enregistrement dans les chiffres indispensables aux données Internet gigantesques d’aujourd’hui. Il est impossible de mettre à l'échelle horizontalement car les bases de données relationnelles suivent un schéma particulier. Bien que la mise à l'échelle verticale semble être la réponse évidente, ce n'est pas le cas. La mise à l'échelle verticale a une limite et, à cette époque et à cet âge, les données collectées quotidiennement via Internet sont tout simplement trop volumineuses pour imaginer que la mise à l'échelle verticale fonctionnerait longtemps. Les contraintes de schéma empêchent également la migration des données vers et depuis différents SGBDR. Ils doivent être identiques ou cela ne fonctionnera pas simplement.
  • Les bases de données non relationnelles ne possèdent aucune structure ou schéma spécifique pour les données stockées, vous ne pouvez pas vous fier à vos données pour un champ donné, car elles ne l’auraient peut-être pas. En l'absence de relations, il est très difficile de mettre à jour les données car vous devrez mettre à jour chaque détail séparément
